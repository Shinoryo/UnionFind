# Union Find 仕様メモ

N個の要素のグルーピングを管理できる構造。

## メソッド一覧

### コンストラクター `__init__(n: int)`

N個それぞれが独立している（どの2つの要素も同じグループではない）Union Findを作成する。

- **パラメータ**: `n` は1以上の整数
- **例外**: `n < 1` の場合は `ValueError` を発生
- **事後条件**:
  - 各要素は独立したグループを形成
  - `group_count()` は `n` を返す
  - すべての要素について `size(i) == 1`

### `union(x: int, y: int) -> bool`

xが所属するグループとyが所属するグループを併合させる。元々xとyが異なるグループに所属しており、併合に成功した場合はTrue、そうでなければFalseを返す。

- **パラメータ**: `x`, `y` は 0 以上 n 未満の整数
- **戻り値**:
  - `True`: xとyが異なるグループに所属しており、併合に成功した場合
  - `False`: xとyが既に同じグループに所属している場合（`x == y` を含む）
- **例外**: `x < 0` または `x >= n` または `y < 0` または `y >= n` の場合は `IndexError` または `ValueError` を発生
- **副作用**:
  - 併合に成功した場合、`group_count()` が 1 減少
  - 併合されたグループのサイズは両グループのサイズの合計となる

### `find(x: int) -> int`

xが所属するグループの代表の要素を返す。

- **パラメータ**: `x` は 0 以上 n 未満の整数
- **戻り値**: xが所属するグループの代表要素（0 以上 n 未満の整数）
- **例外**: `x < 0` または `x >= n` の場合は `IndexError` または `ValueError` を発生
- **保証**:
  - 同じグループに属するすべての要素に対して、同じ代表要素が返される
  - 複数回呼び出しても一貫した代表要素が返される

### `same(x: int, y: int) -> bool`

xとyが同じグループに所属していればTrue、そうでなければFalseを返す。

- **パラメータ**: `x`, `y` は 0 以上 n 未満の整数
- **戻り値**:
  - `True`: xとyが同じグループに所属している場合
  - `False`: xとyが異なるグループに所属している場合
- **例外**: `x < 0` または `x >= n` または `y < 0` または `y >= n` の場合は `IndexError` または `ValueError` を発生
- **保証**:
  - 推移性: `same(x, y)` かつ `same(y, z)` ならば `same(x, z)`
  - 対称性: `same(x, y)` ならば `same(y, x)`
  - 反射性: `same(x, x)` は常に `True`

### `size(x: int) -> int`

xが所属しているグループの要素数を返す。

- **パラメータ**: `x` は 0 以上 n 未満の整数
- **戻り値**: xが所属するグループの要素数（1以上の整数）
- **例外**: `x < 0` または `x >= n` の場合は `IndexError` または `ValueError` を発生
- **保証**:
  - 同じグループに属するすべての要素に対して、同じサイズが返される
  - 初期状態ではすべての要素について `size(x) == 1`

### `group_count() -> int`

全体のグループ数を返す。

- **戻り値**: 現在のグループ数（1以上n以下の整数）
- **保証**:
  - 初期状態では `n` を返す
  - `union` が成功するたびに 1 減少
  - `union` が失敗した場合は変化しない
